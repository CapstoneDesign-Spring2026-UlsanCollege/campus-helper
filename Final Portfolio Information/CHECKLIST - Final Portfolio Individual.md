# Final Portfolio Individual Checklist

Each student completes this checklist.

## A. My Portfolio Page

- [ ] My file exists in `portfolio/08-individual-portfolios/`.
- [ ] My filename uses my real name.
- [ ] My role is clear.
- [ ] My main responsibilities are clear.
- [ ] My strongest contributions are listed.
- [ ] My contribution descriptions are specific.
- [ ] I included 3–5 strong evidence links.
- [ ] My links work.
- [ ] My page is readable.

## B. My Contribution Evidence

- [ ] I linked at least one PR when available.
- [ ] I linked at least one Issue when available.
- [ ] I linked a commit, review comment, screenshot, test result, doc update, or demo when useful.
- [ ] I did not use vague statements such as “I worked on the app.”
- [ ] I explained what I personally did.
- [ ] I can explain the evidence I linked.

## C. My Technical Ownership

- [ ] I chose at least one area I can explain clearly.
- [ ] I can explain what the area does.
- [ ] I can explain where the code or doc lives.
- [ ] I can explain the main flow.
- [ ] I can explain one possible failure.
- [ ] I can explain how the area was tested.
- [ ] I can explain one improvement I would make next.

Possible ownership areas:

- one component,
- one function,
- one route,
- one API endpoint,
- one database table or model,
- one bug,
- one test,
- one setup step,
- one architecture decision,
- one documentation area,
- one AI-assisted change and its review process.

## D. My AI Use

- [ ] I listed the AI tools I used.
- [ ] I explained what AI helped with.
- [ ] I explained what I personally checked.
- [ ] I explained what I personally changed.
- [ ] I explained how I tested or verified the work.
- [ ] I identified one area I still do not fully understand, if applicable.
- [ ] I did not claim AI output as finished work without review.

## E. My Reflection

- [ ] I explained what I learned.
- [ ] I explained what I am proud of.
- [ ] I explained what I should have done better.
- [ ] I explained what I would improve next.
- [ ] I identified one skill I want to continue developing.
- [ ] My reflection is specific and honest.

## F. Presentation Readiness

- [ ] I read the team presentation script.
- [ ] I know the final MVP flow.
- [ ] I know the main bugs and limitations.
- [ ] I know the backup plan.
- [ ] I reviewed the technical-defense question bank.
- [ ] I practiced answering questions aloud.
- [ ] I can answer honestly when I do not know something.
- [ ] I am ready before the presentation begins.

## G. Final Review

- [ ] My portfolio page is complete before **Wednesday, June 17, 2026, at 8:00 AM**.
