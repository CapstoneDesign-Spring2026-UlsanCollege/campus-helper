# FAQ — Final Portfolio

## Do we need to rewrite the whole semester?

No.

Organize and link the work you actually have.

## What if we do not have a Sprint Packet for an early week?

Create the required `WEEK_XX.md` file and add what exists.

Example:

```md
# Week 03

## Materials We Have

- [Planning notes](...)
- [Relevant commit](...)

## Note

We do not have a formal Sprint Packet for this week.
```

## Should we create missing documents now?

Do not create fake historical versions.

You should create the required final handoff docs, such as the final setup guide and QA report.

## Should we include old messy files?

Yes, when they show the real project history.

Use clear filenames and folders.

## Can we link to old files instead of moving them?

Yes.

Move files when easy. Link files when moving would break evidence or create confusion.

## Do we need slides?

No.

Slides are optional.

## Is there a maximum number of slides?

No.

The presentation still has a 15-minute maximum.

## Do we need a presentation script?

Yes.

## Do we need a backup demo video?

Not specifically.

Your team chooses the backup format. Your team is responsible for making the presentation work.

## Can one student create all individual portfolio pages?

Each student is responsible for their own page.

## What counts as evidence?

Good evidence includes:

- PRs,
- Issues,
- commits,
- review comments,
- CI runs,
- screenshots,
- test results,
- docs updates,
- demos,
- and linked planning files.

## What does not count?

- “We worked on it.”
- “AI made it.”
- “Almost done.”
- “It worked before.”
- screenshots with no explanation.

## Can we hide unfinished work?

No.

Document limitations honestly.

## Can we add new features at the last minute?

That is usually a bad idea.

Stabilize the core flow first.

## What if AI wrote a lot of our code?

That is allowed.

Your team must still run it, explain it, test it, debug it, and link evidence.
