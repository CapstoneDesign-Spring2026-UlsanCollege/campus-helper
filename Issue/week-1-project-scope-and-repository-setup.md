# Week 1 Issue - Project Scope and Repository Setup

## Issue Summary

At the beginning of the project, the team needed to turn a broad idea, "Smart Campus Assistant," into a realistic capstone project that could be built and demonstrated during the semester.

## Problem

The first project idea included many possible features:

- class schedules
- campus navigation
- cafeteria menus
- reminders
- AI assistant
- event notifications

The scope was too large for an early sprint, and the repository name was inconsistent across documents.

## Impact

- The team risked building too many incomplete features.
- Documentation referenced different repository names.
- The MVP was not yet clear enough for implementation.

## Investigation

The team reviewed the project idea pitch, team agreement, and project overview. We identified the features that were most important for the first working demo:

- authentication
- dashboard
- AI assistant
- schedule/timetable
- notes and campus service modules

## Resolution

- Standardized the project around **Campus Helper**.
- Updated the repository reference to `campus-helper`.
- Defined a realistic MVP focused on working software.
- Established the team workflow and sprint packet process.
