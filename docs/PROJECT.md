# Project Title

Smart Campus Assistant

---

# Problem Statement

Students often struggle to find important campus information quickly.  
Class schedules, building locations, cafeteria menus, and campus events are often spread across multiple websites or systems.  
This makes it difficult for students to access the information they need efficiently.  
A centralized campus assistant app can make campus life easier by organizing all important information in one place.

---

# Target Users

The main users of this system are:

- University students
- New students unfamiliar with the campus
- Faculty members who need quick access to schedules and events

---

# Project Goal

The goal of this project is to build a web/mobile application that helps students easily access campus information.  
The system will allow users to view class schedules, navigate campus buildings, check cafeteria menus, receive event notifications, and ask questions through an AI assistant.

---

# Core Features (First Version)

- View personal class schedules
- Navigate campus buildings using a simple map
- Check daily cafeteria food menus
- Receive reminders for assignments or events
- Ask questions to an AI campus assistant

---

# Demo Scenario (Very Important)

Step 1  
A student opens the Smart Campus Assistant web app.

Step 2  
The student logs in and sees their class schedule on the dashboard.

Step 3  
The student searches for the IT building and views its campus location.

Step 4  
The student asks the AI assistant:  
"Where is the IT building?" or "When is my next class?"

Step 5  
The system responds with the correct information.

---

# Minimum Viable Product (MVP)

The MVP will be a simple web application where students can log in and access basic campus information.  
The system will display a class schedule, show campus building information, and allow users to check the cafeteria menu.  
A basic AI chatbot will answer simple campus-related questions.  
This will demonstrate the core concept of a centralized campus assistant.
