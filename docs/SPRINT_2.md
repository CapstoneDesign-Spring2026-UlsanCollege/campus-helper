# Sprint 2 - Midterm Preparation Plan

## Sprint Name

Sprint 2: Midterm Demo Readiness

## Purpose

Sprint 2 is the multi-week sprint leading to the Midterm. The goal is not to add random new features. The goal is to make Campus Helper easier to explain, demo, trust, and rehearse.

## Sprint Goal

Prepare a clear Midterm story and a reliable demo that proves Campus Helper is a real working product.

## Midterm Story

### Problem

Students need one place to manage campus information, academic resources, communication, and support. Without a centralized system, important tasks are spread across different tools and become harder to complete quickly.

### User

The primary users are Ulsan College students. Secondary users include new students, student groups, and administrators who need to publish notices.

### Solution

Campus Helper is a full-stack web app that provides authentication, dashboard navigation, AI chat, timetable, notes, marketplace, lost-and-found, friend networking, direct chat, and admin notices.

### Main Demo Flow

1. A student signs in and opens the dashboard.
2. The student asks the AI assistant a campus or study question and sees a streaming answer.
3. An admin publishes a notice that appears in the system.

### Proof

- Working Next.js application
- MongoDB-backed data models
- JWT authentication
- AI SDK streaming chat
- Admin-only notice publishing
- Updated README, sprint packets, and issue logs
- Passing TypeScript and production build checks

### Roadmap Before Final

- Deploy the application to a stable hosted environment.
- Add seed/setup instructions for demo accounts.
- Improve automated testing.
- Improve notification behavior and polish remaining module UI.
- Prepare a recorded backup demo.

## Demo Reliability Plan

### Main Demo

- Run the app locally or from a deployed URL.
- Use prepared student and admin accounts.
- Use a short, rehearsed AI prompt.
- Keep the demo focused on one student workflow plus one admin workflow.

### Backup Demo

- Recorded walkthrough video.
- Narrated screenshots.
- GitHub code walkthrough.
- Build output and documentation evidence.
