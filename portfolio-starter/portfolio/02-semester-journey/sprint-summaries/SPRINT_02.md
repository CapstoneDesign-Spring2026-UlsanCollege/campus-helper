# Sprint 02 Summary

## Sprint Goal

Develop the core functionality of Campus Helper by expanding the platform beyond authentication, implementing major student features, improving frontend-backend integration, and preparing a working MVP for demonstration.

---

## What We Planned

- Create the student dashboard.
- Implement the Notes sharing module.
- Develop the Marketplace module.
- Improve frontend and backend integration.
- Expand database models and API routes.
- Improve UI responsiveness.
- Continue project documentation.
- Prepare a working demo for review.

---

## What We Completed

- Developed the student dashboard.
- Implemented the Notes sharing system.
- Implemented the Marketplace module.
- Connected frontend pages with backend APIs.
- Improved MongoDB database integration.
- Added CRUD functionality for Notes and Marketplace.
- Improved UI layout and navigation.
- Updated project documentation and sprint reports.
- Conducted integration testing for major features.

---

## What Is Still Incomplete

- AI Assistant integration.
- Admin dashboard and notice management.
- Like functionality.
- Real-time notifications.
- Live chat feature.
- Advanced search and filtering.
- Production deployment setup.
- Automated testing.

| Feature or task | Decision | Why |
|---|---|---|
