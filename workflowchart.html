User Registration Flow

📋 Signup Flow Overview

User Registration Process:

  • User enters email, password, and name
  • Frontend validates input (format, strength, matching)
  • POST request sent to /api/auth/signup
  • Backend checks if email already exists
  • Password hashed with bcrypt (10 salt rounds)
  • User document created in MongoDB
  • JWT tokens generated (access + refresh)
  • Tokens stored in HTTP-only cookies
  • User redirected to /dashboard

User Login & Token Refresh Flow

🔐 Login Flow Overview

Key Features:

  • Rate limiting: Max 5 failed attempts per 15 minutes (Redis-backed)
  • User lookup by email from MongoDB
  • Password verification using bcrypt.compare()
  • JWT tokens with different expiration times
  • HTTP-only cookies prevent XSS attacks
  • Access token: 15 minutes (short-lived)
  • Refresh token: 7 days (long-lived)

Token Refresh: When access token expires, the client uses the refresh token to get a new access token without re-entering credentials.

AI Assistant Chat Flow

🤖 AI Chat Flow Overview

Real-time AI Assistance:

  • Load previous conversation history from MongoDB ChatHistory
  • User can optionally attach files (PDFs, images, documents)
  • File upload to Cloudinary via proxy route
  • POST to /api/ai/chat with message and optional fileUrl
  • Backend sends request to OpenAI API (gpt-4) or Google Gemini
  • Uses streaming for real-time token delivery
  • Frontend accumulates tokens and updates UI progressively
  • Full response saved to ChatHistory in MongoDB
  • Conversation persists across sessions

Providers: OpenAI (primary) with fallback to Google Gemini

Direct Messaging Chat Flow

Admin Console - Announcements & User Management

⚙️ Admin Console Overview

Protected Admin-Only Access:

  • Route requires user.role === "admin"
  • Admin verification at /api/admin routes
  • 403 Forbidden for non-admin users

🔔 Publishing Announcements

  • Create announcement with title and content
  • Optional image upload to Cloudinary
  • POST to /api/admin/announcements
  • Automatically create notifications for all users
  • Students see in /dashboard/notifications
  • Toast notifications to logged-in users

👥 User Management

  • View all users in system
  • Search and filter by email
  • Promote users to admin role
  • Revoke admin privileges
  • Suspend/reactivate accounts
  • Changes saved to MongoDB User collection
  • User must re-login to see role changes in JWT
